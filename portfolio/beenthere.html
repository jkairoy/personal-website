Been There

FULL STACK ENGINEERING
Helping trauma victims find their voice with
an anonymous network based off of one way
communication
TECH
React.js, D3, MongoDB,
Node.js
TEAM
2 full stack engineers,
Yusuf Karim and myself
TIME
August 2019 (2 days)

The Goal

To help trauma survivors find comfort in community by creating a safe way to share and recieve trauma narriatives.

The Design

We felt that the best way for messages like this to be communicated was through a message-board-esque design, where posts could be made, found, and commented on. We wanted to avoid triggering our users at all costs, thus, we gave the option to include possibly triggering topics, which would essentially sensor the post until a confirmation had been given.
The most important thing to our team was to make sure that our website was a safe space where victims could not be exploited. Thus, we chose to make communication only possible in one direction to protect users from grooming.
We believed that it was also essential to give users instant feedback and affirmation after their post had been made, thus, if the user opted in, using our tag system, we were able to show our user an animated bubble graph of stories similar to theirs immediately after posting.

The Execution

FRONTEND
The frontend was made using React.js and D3 for the post visualization.
BACKEND
The backend was a very simple NoSQL implementation enabled by MongoDB cloud hosting with client data transfer facilitated by Node.js

Takeaways

This short project was a valuable exploration into the power of data visualization in UX design. Data is powerful, but without the proper layout, it is essentially meaningless. Moreover, this was a great experience in user feedback and in creating a product off of user narrative, as we were able to talk to several trauma victims personally, which profoundly impacted the direction of our design.The project is hosted here if you would like to take a look.
